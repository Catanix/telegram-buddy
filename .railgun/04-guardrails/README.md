# 04-guardrails: Security & Delivery

This layer defines what must be checked before any work is considered complete.

## Purpose
- Enforce security rules and secret handling
- Define mandatory self-review checklists
- Specify commit message and delivery standards
- Prevent common mistakes from reaching production

## Maintenance
Treat every security incident or production bug as a signal: if it could have been caught here, add a guardrail.
