# 01-domain: Business Logic

This layer defines WHAT the product is and how the business speaks about it.

## Purpose
- Establish Ubiquitous Language: strict naming conventions for entities and concepts
- Document data model boundaries and validation limits
- Map core business flows and transactional sequences
- Prevent semantic drift across the codebase

## Maintenance
Own this layer. Product managers, domain experts, and senior engineers should write and update these files.
